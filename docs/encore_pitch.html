1 · THE PROBLEM

游戏 LIVE 的观众,是被动消费的。

主播打出 ace、抢龙、决赛圈最后一击 —— 观众只能看,然后刷条弹幕。 高光发生的那 5 秒,观众的注意力是峰值的,但他们没有任何动作可以"参与"。 礼物是单向的,弹幕是文字的,看完就走。观众的高光时刻情绪没被接住,转化成关注和回访的概率被白白浪费。

~3.5s
高光发生时观众平均互动延迟(刷弹幕到提交)
30s
Encore 给观众的"参与窗口" —— 直接玩一局基于高光的 mini game

2 · WHAT IS ENCORE

把主播的高光时刻,实时重构成观众能玩的 30 秒迷你游戏。
👁

OBSERVE

AI Vision 持续观察 LIVE 画面,识别 kill / clutch / ace / 抢龙 等高光事件。本质是把"主播按高光按钮"自动化。

🧬

STRUCTURE

把高光时刻提取成一份 JSON config:游戏类型、地图主题、敌人数、起始 HP、武器、场景描述。这就是 V2G 的"语言"。

🎮

RECONSTRUCT

客户端在 30 秒内把 config 渲染成一个体素像素风的 mini game,以 bottom sheet 形式弹给当时在场的观众。

4 · THE 3 TEMPLATES (PHASE 1)

不是真生成式,是模板参数化。

Phase 1 (Path B) 的核心决定:AI 不写游戏代码,只产 JSON config。3 个预制模板分别对位 TikTok 游戏 LIVE 的三大内容供给。 覆盖率约 60-70%。

核心 UX 铁律

Encore mini game 以 bottom sheet 形式上滑覆盖,占屏 80%。LIVE 视频依然在播,主播声音依然在响,只是被推到背景层。

  • 永不 pause LIVE,永不静音,永不挤压 main view
  • 玩家任意时刻可以下滑回 LIVE,游戏自动结束
  • 30 秒后自动结束,弹回 LIVE,无侵入
  • 千元机也跑得动:Canvas 2D + 体素方块,无 WebGL
  • UX 状态机 8 个状态(见飞书 §11.7) —— SAMPLING / DETECTED / WARMUP / PLAY / END / REPLAY_PEEK / GIFT / RESET

6 · GIFT ECONOMY

4 层礼物分层,不送礼也能玩。

铁律:禁止"送礼换试玩"。送礼是放大体验,不是入场券。 主播分成 ≥ 普通 LIVE 礼物分成。

ENHANCE
改造你的下一局
"火焰子弹"皮肤、"双倍 HP"开局、"额外一条命"…… 个人增益。
SPOTLIGHT
为高分录像加曝光
让你的录像更可能进入主播 Top 3 锐评池。社交向。
SPONSOR
主播下一局的强化
给主播在下次 LIVE 高光生成的 Encore 加强化 / 主题。礼物流向主播。
LOOT
概率宝箱皮肤
开箱拿稀有装备外观,送回个人收藏。常驻收益。

9 · LOCKED PRODUCT DECISIONS

设计真理在飞书,代码追随文档。
PATH
Path B(模板参数化),不做真生成式。AI 出 JSON config → 路由到 3 模板。
ART
抽象像素风(Roblox / MC 系)。规避所有游戏 IP 风险,千元机兼容。
ENTRY POINT
LIVE-first。M12 之前不做短视频流入口(转化率塌方)。
UX RULE
Bottom sheet 上滑覆盖,永不压缩 LIVE 主画面。8-状态有限状态机。
MONETIZATION
4 层礼物经济(Enhance / Spotlight / Sponsor / Loot)。禁止"送礼换试玩"。
TEMPLATE COVERAGE
3 套模板必备(FPS + MOBA + BR),覆盖约 60-70% 游戏 LIVE 内容。
CREATOR SHARE
Encore 礼物中主播分成 ≥ 普通 LIVE 礼物分成。生态绑定。
DEMO SCOPE
客户端 + 局域网部署。无后端服务(observer.py 是本地代理,不是 backend)。
